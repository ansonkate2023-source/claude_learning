# 報告撰寫（五階段流程）

## 階段一：定義

先問使用者確認（如果 $ARGUMENTS 沒有包含以下資訊）：

1. 報告給誰看？
2. 要回答什麼核心問題？（最多 3 個）
3. 格式要求？（字數、語言、結構）
4. 資料來源？

確認後才進入下一階段。

## 階段二：研究

根據資料來源：
- 本地檔案 → 用 Read / Grep / Glob
- 網路 → 用 WebFetch
- 多來源 → 用 Agent 平行研究

將研究結果寫入 `/tmp/report-research.md`，每個來源附 URL。

## 階段三：大綱

根據研究結果建立大綱，每章標註：
- 一句話說明
- 預估字數
- 引用的資料來源

**輸出大綱後暫停，等使用者確認再繼續。**

## 階段四：撰寫

分段寫入 `/tmp/report-draft.md`：
- 每次寫 1-2 個章節
- 每章控制在 500-800 字
- 數據用 bash 計算，不靠記憶
- 引用標註來源

## 階段五：審查

讀取 /tmp/report-draft.md，逐項檢查：
1. 每章是否回答了大綱問題？
2. 有無沒有根據的斷言？
3. 摘要能否 30 秒看懂結論？
4. 字數是否達標？

列出問題 → 修改 → 搬到正式位置 → git commit + push。
