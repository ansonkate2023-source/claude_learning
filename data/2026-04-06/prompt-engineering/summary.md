# Prompt Engineering 技巧 - 2026-04-06

## 一、基礎原則（官方最佳實踐）

### 1. 清晰直接
把 Claude 當作一位聰明但剛入職的新同事——缺乏你的背景脈絡。指令越精確，結果越好。

**黃金法則：** 把你的 prompt 拿給一位不了解任務的同事看，如果他會困惑，Claude 也會。

```
❌ 建立一個分析儀表板
✅ 建立一個分析儀表板。包含盡可能多的相關功能和互動。超越基礎，建立一個完整的實作。
```

**難度：** 入門

---

### 2. 提供動機與脈絡
解釋「為什麼」比只說「做什麼」更有效。Claude 能從解釋中舉一反三。

```
❌ 絕對不要使用省略號
✅ 你的回覆將由文字轉語音引擎朗讀，所以絕不使用省略號，因為引擎不知道如何發音。
```

**難度：** 入門

---

### 3. 善用範例（Few-shot Prompting）
範例是控制 Claude 輸出格式、語氣和結構最可靠的方式。

**要點：**
- 提供 3-5 個範例效果最佳
- 範例要**相關**（貼近實際用途）、**多樣**（涵蓋邊界情況）
- 用 `<example>` 標籤包裹範例，與指令區隔

**難度：** 入門

---

### 4. 使用 XML 標籤結構化 Prompt
當 prompt 混合指令、上下文、範例和變數輸入時，XML 標籤幫助 Claude 清楚區分各部分。

```xml
<instructions>你的任務指令</instructions>
<context>背景資料</context>
<examples>
  <example>範例 1</example>
  <example>範例 2</example>
</examples>
<input>使用者輸入</input>
```

**難度：** 進階

---

### 5. 給 Claude 一個角色
在 system prompt 中設定角色，能集中 Claude 的行為與語氣。

```
System: 你是一位專精 Python 的程式設計助理。
```

**難度：** 入門

---

### 6. 長文本處理技巧
處理 20k+ token 的大型文件時：

- **長文本放最前面**，查詢放後面（可提升 30% 回應品質）
- 用 XML 包裹多文件結構
- 要求 Claude **先引用相關段落**再執行任務

**難度：** 進階

## 二、15 個 Prompt 模式（社群整理）

來源：Blockchain Council「Prompt Engineering for Claude Mythos: 15 Patterns」

1. **角色扮演法** — 指定專業身份
2. **步驟分解法** — 將複雜任務拆成編號步驟
3. **限制條件法** — 明確字數、格式、語言限制
4. **範例引導法** — 提供 input/output 範例對
5. **反面範例法** — 說明「不要」做什麼
6. **鏈式思考法** — 要求逐步推理後再給答案
7. **自我驗證法** — 要求 Claude 檢查自己的答案
8. **多視角法** — 要求從不同角度分析
9. **摘要優先法** — 先給結論再展開細節
10. **模板填充法** — 提供輸出模板讓 Claude 填寫
11. **對比分析法** — 要求比較多個選項的優劣
12. **情境模擬法** — 設定具體場景進行演練
13. **遞進式提問** — 從簡單到複雜逐步深入
14. **後設認知法** — 要求 Claude 解釋自己的推理過程
15. **品質控制法** — 要求多次迭代改善輸出

## 三、每日實用 Prompt（5 個高頻技巧）

來源：TechRadar「I use these 5 clever Claude prompts every day」

1. **快速摘要** — 貼入長文，要求以 3 個要點摘要
2. **程式碼審查** — 貼入程式碼，要求找出問題並建議改善
3. **決策矩陣** — 描述選項，要求建立比較表格
4. **學習助教** — 要求以簡單類比解釋複雜概念
5. **寫作改善** — 貼入草稿，要求改善清晰度和簡潔性

---

## 四、FORGE 方法

來源：Tom's Guide

將粗略想法轉化為成品的結構化流程：
- **F**rame（框架）— 定義目標和範圍
- **O**utline（大綱）— 建立結構
- **R**efine（精煉）— 迭代改善
- **G**enerate（生成）— 產出完整內容
- **E**valuate（評估）— 檢查品質

## 五、進階技巧

### 控制輸出格式
```
❌ 不要在回覆中使用 markdown
✅ 你的回覆應由流暢的散文段落組成。
```

**重點：** 告訴 Claude「要做什麼」比「不要做什麼」更有效。

### 引導思考行為
Claude 4.6 使用 adaptive thinking，會根據問題複雜度自動調整思考深度。
- 需要深度推理時，使用較高的 effort 設定
- 簡單任務可降低 effort 減少延遲

### 減少幻覺
```
在回答之前，先閱讀相關檔案。絕不對未打開的程式碼做推測。
確保所有回答都有根據，避免幻覺。
```

**難度：** 專家
