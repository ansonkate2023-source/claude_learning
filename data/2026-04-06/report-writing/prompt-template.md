# 報告撰寫 Prompt 樣板

複製以下模板，填入 `___` 的部分，貼給 Claude 即可。

---

## 模板一：完整版（適合重要報告）

```
幫我寫一份報告。

【讀者】
- 誰會看這份報告：___（例如：技術主管 / 客戶 / 團隊成員）
- 他們的技術水平：___（例如：熟悉 AI 但不寫程式 / 完全不懂技術 / 資深工程師）
- 他們最關心什麼：___（例如：成本 / 時間 / 風險 / 可行性）

【核心問題】
這份報告要回答：
1. ___
2. ___
3. ___

【讀者會問的問題】
他們看完最可能問：
1. ___
2. ___
3. ___
4. ___
5. ___
報告的每個章節必須回答至少一個問題。如果某個問題沒被回答到，提醒我。

【格式】
- 語言：___（正體中文 / 英文）
- 字數：___（例如：2000-3000 字）
- 結構要求：___（例如：要有摘要、表格、建議章節）
- 語氣：___（例如：正式 / 口語 / 商業簡報風格）

【資料來源】
- ___（例如：讀取 data/ 目錄下的檔案）
- ___（例如：搜尋網路上關於 XXX 的最新資訊）
- ___（例如：根據我接下來貼的內容）

【特別注意】
- ___（例如：不要用技術術語 / 每個結論要附數據 / 避免提到競品名稱）

請先產出大綱，我確認後再開始寫。
```

## 模板六：需求審查

```
幫我審查以下需求，產出一份需求審查報告。

【需求來源】___（例如：客戶提出 / PM 整理 / 會議記錄）
【專案背景】___（一句話描述這個專案在做什麼）

【需求內容】
（貼上需求文件，或描述需求）
___

【審查面向】
請從以下角度審查每一條需求：

1. **完整性** — 是否有遺漏的情境或邊界條件？
2. **明確性** — 是否有模糊的描述？（「系統應快速回應」→ 快是多快？）
3. **可行性** — 以目前的技術和資源，能做到嗎？難度如何？
4. **一致性** — 需求之間有沒有互相矛盾？
5. **可測試性** — 怎麼驗證這個需求做完了？驗收標準是什麼？
6. **優先級** — 哪些是必須的？哪些是 nice-to-have？

【輸出格式】
每條需求一個區塊：

| 需求 | 狀態 | 問題 | 建議 |
|------|------|------|------|
| 需求描述 | ✅ 清晰 / ⚠️ 待釐清 / ❌ 有問題 | 具體問題 | 改善建議 |

最後附上：
- 遺漏清單：需求文件沒提到但應該要討論的事
- 風險清單：可能導致延期或失敗的需求
- 建議的追問清單：下次會議要問 ___ 的問題

語言：___，格式：條列式，先給大綱。
```

## 模板七：向主管提案

```
幫我寫一份提案，說服主管同意 ___。

【背景】
- 目前的問題 / 痛點：___
- 這個問題造成的影響：___（例如：每週浪費 X 小時 / 客訴增加 X% / 錯誤率 X%）

【提案內容】
我建議：___

【主管的立場】
- 主管最在意的事：___（例如：成本 / 時程 / 人力 / 風險）
- 主管可能的顧慮：___（例如：團隊沒經驗 / 預算不夠 / 正在趕專案）
- 主管的決策風格：___（例如：需要數據說服 / 偏好看 demo / 重視團隊意見）

【主管會問的問題】
1. 為什麼要現在做？不做會怎樣？
2. 要花多少錢 / 多少人 / 多少時間？
3. 有沒有更簡單的替代方案？
4. 風險是什麼？失敗了怎麼辦？
5. 誰負責？怎麼衡量成功？
（如果你知道主管還會問什麼，加在這裡：___）

【寫作要求】
- 第一段就給結論和建議行動，不要鋪陳
- 用數據說話，不要用「我覺得」「應該會」
- 附上成本效益分析（即使是粗估）
- 附上時間表和里程碑
- 附上最小可行方案（如果主管不想一次投入太多）
- ___ 字以內，___（語言）

提案結構建議：
1. 一句話結論（我建議做 X，因為 Y）
2. 問題現況（用數據描述痛點）
3. 提案方案（做什麼、怎麼做）
4. 成本效益（投入 vs 回報）
5. 時間表（里程碑）
6. 風險與備案
7. 最小可行方案（先試小的）
8. 下一步行動（需要主管做什麼決定）

先給大綱。
```

## 使用技巧

1. **一定要填「讀者會問的問題」** — 這是整份報告最重要的輸入，比主題本身還重要
2. **一定要先看大綱** — 結構錯了內容再好也沒用，大綱是唯一的檢查點
3. **不確定讀者會問什麼？** 先問 Claude：
   ```
   我要寫一份 ___ 報告給 ___。
   你覺得他們看完最可能問哪 5 個問題？
   ```
4. **報告太長？** 加一句：「每段控制在 500 字以內，分段寫」
5. **要修改？** 不要重寫，用 Edit：「把第 3 章的結論改成 ___」
