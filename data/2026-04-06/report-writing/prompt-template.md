# 報告撰寫 Prompt 樣板

複製以下模板，填入 `___` 的部分，貼給 Claude 即可。

---

## 模板一：完整版（適合重要報告）

```
幫我寫一份報告。

【讀者】
- 誰會看這份報告：___（例如：技術主管 / 客戶 / 團隊成員）
- 他們的技術水平：___（例如：熟悉 AI 但不寫程式 / 完全不懂技術 / 資深工程師）
- 他們最關心什麼：___（例如：成本 / 時間 / 風險 / 可行性）

【核心問題】
這份報告要回答：
1. ___
2. ___
3. ___

【讀者會問的問題】
他們看完最可能問：
1. ___
2. ___
3. ___
4. ___
5. ___
報告的每個章節必須回答至少一個問題。如果某個問題沒被回答到，提醒我。

【格式】
- 語言：___（正體中文 / 英文）
- 字數：___（例如：2000-3000 字）
- 結構要求：___（例如：要有摘要、表格、建議章節）
- 語氣：___（例如：正式 / 口語 / 商業簡報風格）

【資料來源】
- ___（例如：讀取 data/ 目錄下的檔案）
- ___（例如：搜尋網路上關於 XXX 的最新資訊）
- ___（例如：根據我接下來貼的內容）

【特別注意】
- ___（例如：不要用技術術語 / 每個結論要附數據 / 避免提到競品名稱）

請先產出大綱，我確認後再開始寫。
```

## 模板二：精簡版（適合日常報告）

```
幫我寫一份報告。

給 ___ 看，他們最想知道 ___。
核心問題：___
字數 ___ 字以內，___（語言）。
資料來源：___

他們看完會問：
1. ___
2. ___
3. ___

先給我大綱。
```

## 模板三：週報 / 進度報告

```
幫我寫本週的進度報告。

【讀者】___（主管 / 團隊）
【期間】___ 到 ___

【本週完成的事】
1. ___
2. ___
3. ___

【遇到的問題】
1. ___

【下週計畫】
1. ___
2. ___

讀者會問：
1. 進度正常嗎？有沒有延遲？
2. 需要我幫忙什麼？
3. 下週最重要的事是什麼？

格式：___ 字以內，條列式，正體中文。
```

## 模板四：技術評估報告

```
幫我寫一份技術評估報告。

【評估對象】___（例如：Claude Code 的 skill 測試框架）
【讀者】___，技術水平：___
【評估目的】___（例如：判斷是否適合導入團隊使用）

【評估面向】
請從以下面向評估：
- 功能完整度
- 成熟度 / 穩定度
- 導入成本（時間、人力、金錢）
- 風險和限制
- 與現有方案的比較

【資料來源】___

讀者會問：
1. 這東西能用嗎？成熟度如何？
2. 導入需要多少資源？
3. 風險是什麼？最壞情況？
4. 跟我們現在的做法比，好在哪？
5. 你的建議是什麼？

格式：___字，正體中文，要有比較表格和明確建議。
先給大綱。
```

## 模板五：給非技術客戶的說明

```
幫我寫一份給客戶的說明文件。

【客戶背景】___（例如：零售業老闆，不懂技術）
【要說明的事】___（例如：我們幫他建了一個自動化資料收集系統）

客戶會問：
1. 這對我的生意有什麼好處？
2. 要花多少錢？
3. 什麼時候能用？
4. 安全嗎？我的資料會不會外洩？
5. 我需要做什麼？

寫作要求：
- 完全不用技術術語，用生活化的類比
- 每個功能用「你可以 ___」的句型描述
- 附上 action items：客戶接下來要做的 3 件事
- ___ 字以內，正體中文

先給大綱。
```

---

## 使用技巧

1. **一定要填「讀者會問的問題」** — 這是整份報告最重要的輸入，比主題本身還重要
2. **一定要先看大綱** — 結構錯了內容再好也沒用，大綱是唯一的檢查點
3. **不確定讀者會問什麼？** 先問 Claude：
   ```
   我要寫一份 ___ 報告給 ___。
   你覺得他們看完最可能問哪 5 個問題？
   ```
4. **報告太長？** 加一句：「每段控制在 500 字以內，分段寫」
5. **要修改？** 不要重寫，用 Edit：「把第 3 章的結論改成 ___」
